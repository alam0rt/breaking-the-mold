# Skullmonkeys PC port (`port/`)

A native PC build of the Skullmonkeys decompilation. It reuses the shared game C
in `../src/` unchanged and replaces the PlayStation hardware libraries (PSY-Q:
libgpu / libgte / libspu / libcd / BIOS) with PC backends. The MIPS byte-match
build (`make check` in the repo root) is completely unaffected — the port is
additive and invisible to it.

See the full roadmap in [`../docs/plans/pc-port.md`](../docs/plans/pc-port.md).

## Status

**Phase 0 complete** — the tree compiles 32-bit, links an executable, opens an
SDL2 window + OpenGL 2.1 context, and aborts on the first unimplemented HAL/game
call (`port_game_main`, wired in Phase 2). Phase 1 fills the SPEC/HAL backends;
Phase 2 converts the boot/frame/render spine to functional C.

## Building

Inside the Nix dev shell (`nix develop` / `direnv allow`), which provides the
32-bit toolchain (`port-cc` → `gcc_multi`), 32-bit SDL2/OpenGL, CMake and Ninja:

```bash
cd port
cmake --preset linux-debug        # configure (Ninja); auto-generates the stub TUs
cmake --build --preset linux-debug
./build-linux-debug/skullmonkeys   # opens a window, aborts on first stub (Phase 0)
```

Presets: `linux-debug`, `linux-release`, `linux-debug-make` (Make instead of
Ninja), plus `windows` / `macos` placeholders.

If SDL2 isn't found, the build falls back to a **windowless** boot that still
links and exercises the HAL-abort path (enough to validate the toolchain).

## How it works (the coexistence mechanism)

The MIPS build never defines `TARGET_PC` and never sees `port/`, so it stays
byte-identical. The PC build differs only through a preprocessor pivot injected
by CMake (`-include port_config.h -DTARGET_PC`):

- **`INCLUDE_ASM(...)` → empty** on PC ([`include/port_config.h`](include/port_config.h)
  defines it before `common.h`). Each shelved function's symbol is satisfied by
  a **weak stub**, or by a real functional-C body once written.
- **Register pins / clobber fences** — the decomp's MIPS-only
  `register T x asm("$N")` pins and `__asm__ volatile("" ::: "$N")` fences are
  swept by [`../tools/portize.py`](../tools/portize.py) to `PSX_REG("$N")` /
  `PSX_CLOBBER("$N")`, which expand to the identical tokens on MIPS (byte-neutral)
  and to nothing on x86.
- **`_autostubs.c`** (generated): a weak `void NAME(void){ port_stub("NAME"); }`
  for every asm-only function. Weak → a real strong body overrides it.
- **`_autoglobals.c`** (generated): weak backing storage for every absolute-address
  global (from `../symbol_addrs.txt` + `asm("D_…")` aliases). Weak → a real C
  definition overrides it.
- Both generated by [`../tools/gen_port_stubs.py`](../tools/gen_port_stubs.py),
  re-run automatically at CMake configure time.

As functions byte-match (or get functional-C bodies), the strong definition
shadows the weak stub — the port strictly improves with no divergence and
nothing to hand-sync.

## Layout

```
port/
  CMakeLists.txt        native 32-bit build; globs ../src + spec/ + decomp/
  CMakePresets.json     linux/windows/macos presets (Ninja default)
  include/
    port_config.h       TARGET_PC pivot: 32-bit assert, INCLUDE_ASM no-op
    psyq_pc.h           PC reconstruction of PSY-Q TYPES (32-bit layouts)
    LIB{API,GTE,GPU,CD,SPU,ETC,PAD}.H  thin shims -> psyq_pc.h
    port_runtime.h      port_stub / port_log / port_panic / port_game_main
  spec/                 SPEC_PSXPC layer — hardware REPLACED on PC
    port_runtime.c      port_stub(), logging, weak port_game_main
    host_main.c         SDL window + real main(); drives the game boot
    hal_stubs.c         weak aborting bodies for the whole PSY-Q HAL surface
    (gpu_gl.c, gte.c, spu_sdl.c, cd_files.c, pad_sdl.c, bios.c — Phase 1)
  decomp/
    _autostubs.c        GENERATED weak function stubs
    _autoglobals.c      GENERATED weak global backing storage
```

## Invariants

- **Never regress the byte-match.** Every port change must keep `make check`
  green in the repo root. The pin/fence sweep expands byte-identically on MIPS.
- **32-bit is load-bearing**, not a preference: the engine overlays pointers at
  fixed struct offsets and strides 8-byte callback slots. A `_Static_assert` in
  `port_config.h` hard-fails a 64-bit build.
